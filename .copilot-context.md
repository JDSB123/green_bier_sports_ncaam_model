# Project Context for AI Assistants

## Tech Stack (Multi-Language Project)

This is a **multi-language NCAA basketball betting model** using:

- **Go** - `services/ratings-sync-go/` - Syncs Barttorvik ratings data
- **Rust** - `services/odds-ingestion-rust/` - Syncs odds from The Odds API
- **Python** - `services/prediction-service-python/` - Core ML prediction engine
- **R** - `scripts/extract_r_package_mappings.R` - Data analysis scripts
- **TypeScript/React** - `services/web-frontend/` - Web UI

## Required Tools/Extensions

- **Go** compiler and tooling
- **Rust** compiler (cargo)
- **Python 3.x** with dependencies
- **R** with languageserver package (for IDE features)
- **Node.js/npm** for frontend
- **Docker** and Docker Compose for containerization
- **PostgreSQL** database

## Architecture Pattern

Single-container deployment with embedded binaries:
- All services run inside the prediction-service container via `services/prediction-service-python/run_today.py`
- Go and Rust binaries are compiled and embedded
- Python orchestrates the pipeline
- Secrets managed via `secrets/` directory (NO .env fallbacks)

## Key Files to Reference

- `README.md` - Main entry point and usage
- `services/prediction-service-python/run_today.py` - Single command to generate picks (manual-only mode)

## Important Notes

- This is NOT a Python-only project
- All languages are actively used in production
- Do NOT suggest removing Go/Rust/R components
- Docker secrets are REQUIRED (no localhost fallbacks)
